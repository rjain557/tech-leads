## Scope of work

### System Assessment

**Problem / context:**

The first phase will involve a thorough assessment of the current 3CX setup to understand existing configurations and the environment where it operates.

**Strategy:**

Conduct a comprehensive review of the current system.

**Objectives:**

To assess current system configurations and potential issues that may arise during the upgrade process.

**General requirements:**

Access to the current system, documentation, and space for a temporary test environment.

**Reporting & monitoring:**

Daily updates during the assessment phase.

**Assumptions:**

Current system configurations and hardware are adequately documented and accessible.

### Upgrade Implementation

**Problem / context:**

This phase will detail the steps to safely upgrade the system from v18 to v20 while ensuring minimal downtime.

**Strategy:**

Backup the existing system, perform the upgrade, and verify functionality post-upgrade.

**Objectives:**

Successfully upgrade the 3CX system without significant disruption.

**General requirements:**

Scheduled downtime with users notified in advance.

**Reporting & monitoring:**

Real-time reporting during the upgrade process.

**Assumptions:**

There will be a backup in place to restore the previous version if necessary.

### Post-Upgrade Support

**Problem / context:**

Provide support after the upgrade, addressing any user issues and ensuring system stability.

**Strategy:**

Organize training sessions for users and provide ongoing technical support.

**Objectives:**

To ensure a successful transition for users and optimal system usage.

**General requirements:**

Access to user feedback about the new functionalities.

**Reporting & monitoring:**

Feedback surveys from users to monitor satisfaction levels.

**Assumptions:**

Users will require training and assistance adjusting to new features.
