## Scope of work

### Planning Phase

**Problem / context:**

This phase involves assessing the current environment, identifying user requirements, and creating a detailed migration plan.

**Strategy:**

Utilizing Microsoft’s best practices for migration, ensuring data integrity and user training.

**Objectives:**

To develop a comprehensive migration strategy that addresses needs for email accounts and data storage.

**General requirements:**

Access to existing GSuite accounts, identification of key stakeholders, and required approvals for the migration.

**Assumptions:**

All required M365 licenses will be procured by Boozdna.

### Migration Phase

**Problem / context:**

Executing the migration plan which includes mailbox and data migrations.

**Strategy:**

Using automated migration tools provided by Microsoft and ensuring backup of all data prior to migration.

**Objectives:**

To ensure a seamless transition of emails and files with minimal disruption to users.

**General requirements:**

Ensure user communication regarding the migration timeline and expected changes.

### Post-Migration Phase

**Problem / context:**

This final phase involves verifying the successful migration of data and providing support to users.

**Strategy:**

Providing user training and support for troubleshooting post-migration issues.

**Objectives:**

Ensure all users have access to their emails and files in Microsoft 365.

**General requirements:**

Monitoring and user feedback collection to address any issues post-migration.
