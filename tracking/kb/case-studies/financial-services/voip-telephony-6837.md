## Scope of work

### Pre-Upgrade Assessment and Planning

**Problem / context:**

This phase involves assessing the current system, planning the upgrade, and preparing necessary documentation.

**Strategy:**

Conduct a thorough assessment and devise a comprehensive upgrade strategy.

**Objectives:**

To clearly identify requirements and create a detailed plan for the upgrade execution.

**General requirements:**

Inventory of current configurations and user requirements must be completed.

**Reporting & monitoring:**

Weekly updates to the client regarding progress and findings.

**Assumptions:**

The current system hardware is adequate for version 20.

### Upgrade Execution

**Problem / context:**

This phase involves executing the upgrade of the 3CX system.

**Objectives:**

To successfully upgrade the system to version 20 with minimal disruption.

**General requirements:**

A rollback plan must be in place in case of critical issues.

**Assumptions:**

All necessary resources and access are available during the upgrade.

### Post-Upgrade Testing & User Training

**Problem / context:**

After the upgrade, test the system functionality and conduct training for users.

**Strategy:**

Conduct comprehensive testing followed by interactive user training.

**Objectives:**

To ensure users are proficient with the new features and the system operates as expected after the upgrade.

**General requirements:**

Sufficient documentation and resources for user training must be provided.

**Assumptions:**

Users are available for training sessions.
