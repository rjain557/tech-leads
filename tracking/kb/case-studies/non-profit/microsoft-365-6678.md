## Scope of work

### Planning

**Problem / context:**

This phase involves preparing for the migration, ensuring all prerequisites are met, and creating a detailed migration plan.

**Strategy:**

Verify admin credentials.Purchase necessary MigrationWiz licenses.Establish communication with GoDaddy support.Create a detailed timeline for the migration process.

**Objectives:**

To create a comprehensive migration plan that addresses all potential risks.To ensure all prerequisites are in place for a smooth migration.

**General requirements:**

Admin credentials for both GoDaddy and Microsoft O365.MigrationWiz licenses for user mailboxes and any shared mailboxes.

**Technical requirements:**

Admin access to the GoDaddy-Hosted Office 365 and Microsoft O365 accounts.MigrationWiz tool setup.

**Assumptions:**

All necessary licenses are purchased before the migration.Adequate communication with GoDaddy support is established for domain release post-migration.

**Success criteria:**

Ensure all pre-migration steps are completed.Confirm readiness to proceed to the Migration phase.

### Migration

**Problem / context:**

This phase involves the actual migration of mailboxes from GoDaddy-Hosted Office 365 to Microsoft O365.

**Strategy:**

Perform pre-stage migration to transfer initial data.Conduct full (delta) migration to ensure all changes are captured.Verify data integrity post-migration.

**Objectives:**

To migrate all 10 mailboxes without data loss.To ensure all mailboxes are fully functional post-migration.

**General requirements:**

Access to MigrationWiz.Verified admin credentials.

**Technical requirements:**

Pre-stage migration and full (delta) migration pass using MigrationWiz.Ensure all data is transferred and verified.

**Assumptions:**

Pre-migration planning and setup are completed successfully.Adequate bandwidth for data transfer is available.

**Success criteria:**

Verify all mailboxes are migrated successfully.Confirm no data loss or corruption.

### Post-Migration Support

**Problem / context:**

This phase provides support after the migration to address any issues and ensure smooth operation.

**Strategy:**

Monitor email system for any issues.Provide user training and support as needed.Address any issues related to email functionality.

**Objectives:**

To resolve any post-migration issues quickly.To ensure users are comfortable with the new system.

**General requirements:**

Post-migration support team availability.Access to both old and new email systems for troubleshooting.

**Technical requirements:**

Tools for monitoring email functionality.Support resources for addressing user issues.

**Assumptions:**

Migration is completed successfully.Users may require assistance during the transition.

**Success criteria:**

Confirm all issues are resolved.Ensure user satisfaction with the new email system.
