# Migrating the client from On-premise Active Directory to Azure Active Directory

- **Industry:** Other
- **Service theme:** cloud-migration
- **Engagement date:** 2025-11-03
- **Estimated effort:** 44 hrs (1232: 44h)

## Client context

This proposal outlines the process to migrate the client’s current on-premise Active Directory environment, which consists of 21 users, to Azure Active Directory (AAD). The scope includes unjoining each desktop PC from the existing domain, joining AAD, and transferring user profiles. Additionally, we will ensure that OneDrive and Outlook email functionalities remain intact post-migration, conduct necessary testing and validation, and properly decommission old servers.

## Scope of work

### Project Preparation

**Problem / context:**

This phase will involve assessing the current environment and preparing for the migration to Azure AD.

**Objectives:**

To gather all necessary data for the migration and ensure all components are ready.

**General requirements:**

Access to the existing Active Directory, Azure portal, and user endpoints.

**Assumptions:**

The existing Active Directory is functioning properly and all user accounts are active.

### Environment Migration

**Problem / context:**

This phase will handle the physical migration of user devices from the on-premise AD to Azure AD.

**Objectives:**

Successfully migrate all users and their configurations to Azure AD.

**General requirements:**

Stable internet connection for users to join Azure AD successfully.

**Assumptions:**

All user data is backed up prior to migration and users will be instructed on how to log into Azure AD after migration.

### Validation and Testing

**Problem / context:**

This phase involves testing the functionality of user accounts and applications post-migration.

**Objectives:**

Ensure all services, particularly OneDrive and Outlook, are functioning as expected.

**General requirements:**

Access to all migrated user accounts and applications for testing purposes.

**Assumptions:**

Users have logged into their accounts successfully post-migration and can access required resources.

### Decommissioning and Documentation

**Problem / context:**

This phase includes decommissioning the old server environment and documentation of the new setup.

**Objectives:**

To properly document the migration process and securely decommission legacy servers.

**General requirements:**

Full access to the old servers for decommissioning.

**Assumptions:**

All user data has been migrated successfully and is operational in Azure AD.
