# Migrate from On-Premise Active Directory to Azure Active Directory

- **Industry:** Other
- **Service theme:** cloud-migration
- **Engagement date:** 2026-02-26
- **Estimated effort:** 50 hrs (1232: 50h)

## Client context

This proposal outlines the planned migration from the current on-premise Active Directory to Azure Active Directory for the client. With 21 desktop users whose profiles require updating, this migration will enhance flexibility, scalability, and reduce infrastructure costs associated with maintaining on-premise systems. The migration is divided into evaluation, design, implementation, and decommissioning phases, ensuring a thorough and organized transition with minimal disruption.

## Scope of work

### Evaluation

**Problem / context:**

This phase involves assessing the current Active Directory environment to identify specific needs and any potential issues before migration.

**Strategy:**

Leverage existing documentation and perform hands-on assessments of current systems.

**Objectives:**

To complete a comprehensive evaluation report that guides the migration strategy.

**General requirements:**

Access to all current Active Directory configurations and user accounts.

**Assumptions:**

The existing environment is stable and meets system requirements for Azure Active Directory.

### Design

**Problem / context:**

This phase entails designing the new Azure Active Directory environment tailored for the client's needs.

**Strategy:**

Utilize best practices in Azure AD implementations and customize the design based on the client's specific requirements.

**Objectives:**

To create a comprehensive design document that outlines structure, roles, and policies.

**General requirements:**

Identification of user roles and access levels in the new Azure directory.

**Assumptions:**

Users will embrace the new Azure platform and any necessary training will be provided.

### Implementation

**Problem / context:**

This phase focuses on the actual migration of user accounts from the on-premise system to Azure Active Directory.

**Strategy:**

Use automated tools for migration to minimize downtime and ensure accuracy of user data transfer.

**Objectives:**

To successfully migrate all user accounts and profiles to Azure Active Directory.

**General requirements:**

Complete user accounts setup and appropriate licenses for Azure AD.

**Assumptions:**

All users will have access to the necessary endpoints and tools during migration.

### Decommissioning Old Server

**Problem / context:**

This phase involves shutting down and safely decommissioning the old on-premise Active Directory server.

**Strategy:**

Ensure all data is backed up and validated, followed by compliant server disposal.

**Objectives:**

To securely decommission the old Active Directory server without data loss.

**General requirements:**

Documentation for data retention and server decommissioning processes.

**Assumptions:**

The new Azure Active Directory is fully operational and all data is verified before decommissioning.
